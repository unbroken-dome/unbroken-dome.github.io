HelmExtension

interface HelmExtension : HelmExecProvider, GlobalHelmOptions

The main Helm DSL extension, accessible using the helm { ... } block in build scripts.

Properties

debug

 abstract val debug: Property<Boolean>

Indicates whether to use the verbose output (--debug flag) when invoking commands.

executable

 abstract val executable: Property<String>

The name or path of the Helm executable. The PATH variable is taken into account, so this can just be "helm" if the Helm client is installed in a suitable location.

home

 abstract val home: DirectoryProperty

The path of the Helm home directory. If this provider has a value, it will be used to set the HELM_HOME environment variable for each Helm invocation.

host

 abstract val host: Property<String>

Address of Tiller, in the format host:port.

kubeConfig

 abstract val kubeConfig: RegularFileProperty

Path to the Kubernetes configuration file.

kubeContext

 abstract val kubeContext: Property<String>

Name of the kubeconfig context to use.

outputDir

 abstract val outputDir: DirectoryProperty

Base output directory for Helm charts.

timeoutSeconds

 abstract val timeoutSeconds: Provider<Int>

Time in seconds to wait for any individual Kubernetes operation (like Jobs for hooks). Default is 300.
