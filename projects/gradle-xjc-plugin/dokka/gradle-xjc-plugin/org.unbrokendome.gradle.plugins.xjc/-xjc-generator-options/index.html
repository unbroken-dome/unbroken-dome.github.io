XjcGeneratorOptions

interface XjcGeneratorOptions (source)

Flag-style options for influencing the XJC generator behavior.

Properties

contentForWildcard

If true, generates content property for types with multiple xs:any derived elements (which is supposed to be correct behavior).

abstract val contentForWildcard: Property<Boolean>

docLocale

The documentation locale to be used when running XJC. This may influence the language of documentation comments in XJC-generated files.

abstract val docLocale: Property<Locale>

enableIntrospection

If true, fix getter/setter generation to match the Bean introspection API.

abstract val enableIntrospection: Property<Boolean>

encoding

The encoding to use for generated files.

abstract val encoding: Property<String>

extension

If true, enable JAXB vendor extensions.

abstract val extension: Property<Boolean>

noFileHeader

If true, suppresses the generation of a file header comment that includes some note and time stamp.

abstract val noFileHeader: Property<Boolean>

packageLevelAnnotations

If true, generate package-level annotations into package-info.java files. If false, those annotations are internalized into the other generated classes.

abstract val packageLevelAnnotations: Property<Boolean>

readOnly

If true write-protect the generated Java source files.

abstract val readOnly: Property<Boolean>

strictCheck

If true, perform strict schema validation.

abstract val strictCheck: Property<Boolean>

targetVersion

The version of the JAXB specification to target.

abstract val targetVersion: Property<String>
